# Brainstorming

## Overview
Do NOT jump into writing code when given a feature request. First, act as a senior engineer brainstorming with a colleague. Get clarity, propose a design, and get sign-off.

## Workflow

1. **Clarify Intent**: 
   - Ask clarifying questions.
   - What is the actual goal of this feature?
   - What are the constraints?

2. **Propose and Explore Alternatives**:
   - Don't just pick the first idea. Propose 2-3 logical ways to implement it.
   - Discuss trade-offs (Performance vs Complexity vs Maintainability).
   - Wait for the human partner to choose or iterate on the options.

3. **Present a Spec/Design**:
   - Write out a clear, high-level specification.
   - What components will change?
   - What APIs are needed?
   - What database schemas are affected?
   
4. **Wait for Validation**:
   - Present the spec in digestible chunks.
   - Do NOT start writing the implementation code until the human partner says "yes, looks good" or "go ahead."
